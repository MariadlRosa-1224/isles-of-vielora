- The idea of using a database is to store a link between each GM, and the user participating in a quest, a personalized story, etc., so each can keep track of what they are doing. The main idea now is for the GM to have a list of different requests they can accept. Once a GM request is accepted, for example, it will disappear from the list or be updated to show as accepted. And when accepted, the moderator is essentially committed to being the person who will fulfill that request for the user. Once cleared, the user will close the GM request and call it fulfilled. And the gm will be asked to summit the links to their Responses into the bot for automatic word count

- **Database Purpose:**
      *   Store links between GMs and users in quests/stories.
      *   Enable tracking of user progress.
  **GM Request System:**
      *   GM has a list of requests to accept.
      *   Acceptance:
          *   Request disappears or is marked as accepted.
          *   GM is committed to fulfilling the request.
  **Request Completion:**
      *   User closes the request as fulfilled.
  **Word Count Automation:**
      *   GM submits response links to the bot.
      *   Bot automatically calculates word count.
